     Plan: Connect ZTech_Agents to Mission-Control

     Context

     Mission-control currently uses hardcoded mock data (3 departments, 6 teams,
      9 agent assignments) in src/lib/mock-org-data.ts. The ZTech_Agents repo at
      /home/rahmanwolied/Wolied/z/ZTech_Agents/ contains the real organizational
      structure: 17 departments, ~100 teams, ~600+ agents, each with rich
     metadata files (AGENT.md, IDENTITY.md, SOUL.md, USER.md). The goal is to
     replace mocks with live filesystem scanning so changes to ZTech_Agents are
     automatically reflected.

     Phase 1: Scanner Foundation

     1a. Add env var to .env.example

     ZTECH_AGENTS_PATH=/path/to/ZTech_Agents

     1b. Create src/lib/ztech-scanner.ts (server-only)

     Core module that scans the ZTech_Agents directory at runtime:

     - stableId(relativePath: string): number - djb2 hash producing
     deterministic positive 32-bit int from directory path. IDs are stable
     across restarts.
     - parseAgentMd(content: string) - extracts metadata from AGENT.md's
     markdown table format (| **Key** | Value |)
     - scanZTechAgents(): ScanResult - walks 3 levels:
       - Level 1: departments (top-level dirs, skip _skills_library, .git,
     README.md)
       - Level 2: teams (subdirs within each department)
       - Level 3: agents (subdirs within each team, parse AGENT.md for
     name/role/skills)
     - getOrgData(): ScanResult - cached wrapper with 30s TTL
     - buildDocTree(basePath: string): DocFile[] - builds nested doc tree from
     filesystem for docs panel
     - readDocContent(relativePath: string): string - reads file content with
     path traversal protection

     Returns: { departments, teams, orgAgents, agentTeamAssignments }

     - Department names: Software_Engineering -> Software Engineering
     - Colors: deterministic from 18-color Catppuccin palette by sorted index
     - Teams inherit department color
     - Empty/missing AGENT.md: fallback to directory name
     - Path validation: all file reads validated to stay within
     ZTECH_AGENTS_PATH
     - Graceful fallback: if ZTECH_AGENTS_PATH unset/invalid, returns empty
     arrays

     1c. Add OrgAgent interface to src/store/index.ts

     export interface OrgAgent {
       id: number; name: string; role: string;
       department_id: number; team_id: number; dir_path: string;
       skills: string[]; deliverables: string[]; kpi?: string;
       status: 'idle'; created_at: number; updated_at: number;
     }

     Add orgAgents: OrgAgent[] and setOrgAgents to store.

     Phase 2: API Route Migration (10 files + 2 new)

     Replace all import { MOCK_* } from '@/lib/mock-org-data' with import { 
     getOrgData, buildDocTree, readDocContent } from '@/lib/ztech-scanner'

     Department routes:
     - GET /api/departments -> getOrgData().departments
     - GET /api/departments/[id] -> find by id
     - GET /api/departments/[id]/teams -> filter teams by department_id
     - GET /api/departments/[id]/docs -> buildDocTree() for department dir
     - GET /api/departments/[id]/docs/content -> readDocContent() with path
     traversal check

     Team routes:
     - GET /api/teams -> getOrgData().teams
     - GET /api/teams/[id] -> find by id
     - GET /api/teams/[id]/members -> filter assignments + enrich with orgAgent
     data
     - GET /api/teams/[id]/docs -> buildDocTree() for team dir
     - GET /api/teams/[id]/docs/content -> readDocContent()

     New routes:
     - GET /api/org/agents -> all org agents
     - GET /api/org/assignments -> all agent-team assignments

     Phase 3: UI Migration (3 files)

     3a. departments-panel.tsx

     - Remove import { MOCK_* } from '@/lib/mock-org-data'

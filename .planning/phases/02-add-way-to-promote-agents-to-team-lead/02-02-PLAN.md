**3. Implement `promoteToLead`** (near `assignAgentToTeam` implementation around line 1292):

```typescript
promoteToLead: async (agentId, teamId) => {
  try {
    const response = await fetch(`/api/teams/${teamId}/assignments`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ agent_id: agentId, role: 'lead' }),
    })
    if (!response.ok) {
      console.error('Failed to promote to lead:', await response.text())
      return false
    }
  } catch (err) {
    console.error('Failed to promote to lead:', err)
    return false
  }

  // Refresh OrgSnapshot cache so useOrgData sees the updated state
  try {
    await fetch('/api/org/scan?force=true')
  } catch {
    // Non-fatal: local state update below will still reflect the change
  }

  // Update local state on success (demote old lead, promote new)
  set((state) => {
    const updated = state.agentTeamAssignments.map((a) => {
      if (a.team_id === teamId && a.role === 'lead' && a.agent_id !== agentId) {
        return { ...a, role: 'member' as const }
      }
      if (a.agent_id === agentId && a.team_id === teamId) {
        return { ...a, role: 'lead' as const }
      }
      return a
    })
    return { agentTeamAssignments: updated }
  })
  return true
},
```

**4. Implement `setDepartmentLead`**:

```typescript
setDepartmentLead: async (deptId, agentId) => {
  try {
    const response = await fetch(`/api/departments/${deptId}/lead`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ agent_id: agentId }),
    })
    if (!response.ok) {
      console.error('Failed to set department lead:', await response.text())
      return false
    }
  } catch (err) {
    console.error('Failed to set department lead:', err)
    return false
  }

  // Refresh OrgSnapshot cache so useOrgData sees the updated state
  try {
    await fetch('/api/org/scan?force=true')
  } catch {
    // Non-fatal: local state update below will still reflect the change
  }

  // Update local state on success
  set((state) => ({
    departments: state.departments.map((d) =>
      d.id === deptId ? { ...d, manager_agent_id: agentId ?? undefined } : d
    ),
  }))
  return true
},
```

**Important:** Leave the existing synchronous `assignAgentToTeam` action unchanged. It is still used by `handleAddMembers` for in-session member additions. The new `promoteToLead` is a separate action specifically for API-backed role changes.

**Why the org refresh:** After the API call succeeds, the server-side `invalidateOrgSnapshot` clears the cache, but the client's `useOrgData` hook still holds the old snapshot. Calling `GET /api/org/scan?force=true` ensures the next `useOrgData` poll returns fresh data including the updated assignments and `manager_agent_id`. This prevents stale state in the embedded chat tabs which resolve leads from the org snapshot.

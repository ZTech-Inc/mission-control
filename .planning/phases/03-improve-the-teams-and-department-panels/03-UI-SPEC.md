## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1`, `gap-1`) |
| sm | 8px | Compact row padding, button inner gap (`px-2`, `gap-2`) |
| md | 16px | Default card/section padding (`px-4`, `gap-4`) |
| lg | 24px | Card interior section breaks (`py-6`, `gap-6`) |
| xl | 32px | Panel column gaps, split-view gutters |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Not used in panels; reserved for page-level |

Exceptions:
- Tab bar height: `py-2` (8px) top/bottom — established by existing panel breadcrumb bars
- Agent roster item: `px-3 py-2` (12px/8px) — matches existing `teams-panel.tsx` member row pattern
- Stat card cells: `px-4 py-3` — matches existing stat card pattern in `teams-panel.tsx` line 131
- Touch targets for tab buttons: minimum 28px height (`h-7`) — established by `Button` size `xs`

**Source:** `teams-panel.tsx` class audit, `button.tsx` size variants

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label / Meta | 11px (`text-[11px]`) | 400 regular | 1.4 |
| Micro / Badge | 10px (`text-[10px]`) | 400 regular, uppercase + `tracking-wide` | 1.2 |
| Heading / Card title | 18px (`text-lg`) | 600 semibold | 1.2 |

All panel text uses `font-mono`. The font-sans stack is reserved for prose content outside panels.

**Source:** `teams-panel.tsx` and `departments-panel.tsx` class audit — existing pattern uses `text-sm font-mono`, `text-[11px] font-mono`, `text-[10px] font-mono uppercase tracking-wide`, `text-lg font-semibold font-mono`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--surface-0))` — dark: `215 27% 4%` | Panel body background, row backgrounds, chat pane |
| Secondary (30%) | `hsl(var(--surface-1))` — dark: `222 35% 7%` and `hsl(var(--surface-2))` — dark: `220 30% 10%` | Section cards, roster cards, sidebar nav, tab bar |
| Accent (10%) | `hsl(var(--primary))` — dark: `187 82% 53%` (void-cyan) | See reserved list below |
| Destructive | `hsl(var(--destructive))` — dark: `0 72% 51%` | Remove agent from team, delete doc, demote lead — inline confirmation only |

Accent (`hsl(var(--primary))` / void-cyan) reserved for:
1. Selected agent highlight ring in the inline agent details card roster (selected row border)
2. Selected agent highlight in Chat tab agent selector (active chip/card border)
3. "New Team" button — solid `bg-primary text-primary-foreground` variant (Button `default`)
4. "New Department" button — solid `bg-primary text-primary-foreground` variant (Button `default`)
5. "Add Member" (create agent) button — solid `bg-primary text-primary-foreground` variant (Button `default`)
6. "Hire a Manager" button — solid `bg-primary text-primary-foreground` variant (Button `default`)
7. Team lead badge label text (`text-primary`) — matches existing `text-primary` pattern in `departments-panel.tsx`
8. Focus ring (`ring` token, which maps to `--primary`)

**Source:** `globals.css` dark theme tokens; `departments-panel.tsx` OverviewMetric `tone='accent'` uses `text-primary`; `teams-panel.tsx` line 360 uses `border-primary/40` for drag overlay.

## Interaction Contracts

### Agent selection in roster (Teams Overview)
- Click agent row: update `InlineAgentDetailsCard` to show selected agent's data. No modal, no navigation.
- Selected row: `border-primary/60 bg-[hsl(var(--surface-2))]` ring. All other rows revert to `border-border/50 bg-[hsl(var(--surface-0))]`.
- On mount: pre-select team lead. If no lead, card shows empty state.
- Keyboard: `ArrowUp` / `ArrowDown` navigate roster; `Enter` confirms selection. Tab order follows DOM order.

### Agent selector in Chat tab
- Chip strip replaces any dropdown. One chip per team member.
- Active chip: `border-primary/60`, agent name visible.
- Click chip: updates `EmbeddedChat` conversation target. Persists as `selectedChatAgentId` in component state shared across tab switches (lifted to `TeamDetail` component state).
- Default on mount: team lead chip pre-selected.

### "New Team" button
- Placement: panel header row, right-aligned, alongside existing controls.
- Variant: `Button default size sm`.
- Opens an inline form (not modal) below the panel header OR a `<dialog>` modal — Claude's discretion on exact container. Form fields: team name (required), description (optional), department auto-populated.
- On submit: POST to team creation API, write directory to filesystem, refresh org data.

### "New Department" button
- Same placement pattern as New Team (panel-level header, right-aligned).
- Variant: `Button default size sm`.
- Modal with: department name (required). On submit: POST to dept creation API + filesystem write.

### "Add Member" button (Teams Members tab)
- Opens `CreateAgentModal` — single-page scroll form, not a wizard.
- Form layout: two-column grid for short fields (name, role, model), full-width textareas for identity files.
- Submit: "Create Agent" label on Button `default`.
- Cancel: Button `ghost` or `outline`.

### "Hire a Manager" button (Department Overview)
- Variant: `Button default size sm`.
- Opens the same `CreateAgentModal` but pre-sets creation path to `ZTech_Agents/<dept>/MANAGER/`.

### "Add Doc" / "New Doc" button (OrgDocsPanel)
- Variant: `Button outline size xs`.
- Prompts for filename input (inline text field, not modal). Press Enter or click "Create" to write `<name>.md` to the `docs/` directory.
- On success: refresh doc list.

### Destructive actions
All destructive actions use inline confirmation pattern (established in Phase 2):
1. First click shows a confirmation row with warning copy and a `Button destructive size xs` confirm button + `Button ghost size xs` cancel button.
2. No `window.confirm()` or modals for destructive actions.

## Accessibility Contract

- All interactive agent roster rows: `role="button"` or `<button>` element; `aria-pressed` for selected state.
- Agent chip selector: `role="radiogroup"` with each chip as `role="radio"` and `aria-checked`.
- Vertical tab sidebar: `role="tablist"` orientation `"vertical"`; each tab `role="tab"` with `aria-selected`; tab panel `role="tabpanel"` with `aria-labelledby`.
- Modal dialogs: `role="dialog"`, `aria-modal="true"`, focus trapped on open, focus returns to trigger on close.
- "New Team", "New Department", "Add Member", "Hire a Manager" buttons: descriptive `aria-label` if icon-only fallback is used.
- Destructive confirmation buttons: `aria-label` includes the entity name ("Remove Aegis from Ops Team").
- Color contrast: `text-muted-foreground` on `surface-0` background meets 3:1 minimum for non-body text; `text-foreground` on surfaces meets 4.5:1.

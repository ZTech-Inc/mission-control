
    **Rendering logic per space:**

    **departments space:**
    - Call `getDepartmentChatEntities(departments, agents)`
    - If no departments exist: EmptyState with title="No departments" subtitle="Create departments in the Departments panel"
    - If departments exist but none have leads: EmptyState with title="No department leads assigned" subtitle="" (still show all items as disabled)
    - Actually: show ALL departments. Those without `leadAgent` render as disabled (`opacity-50`, no onClick, show "No lead assigned" where lead name would go)
    - Each item: `<div role="option" aria-selected={activeConversation === entity.conversationId}>`
      - Layout: `flex items-start gap-2 px-3 py-2 border-b border-border/30 cursor-pointer` (or `cursor-default opacity-50` if no lead)
      - Left: color dot `<span className="w-2.5 h-2.5 rounded-full mt-1 flex-shrink-0" style={{ backgroundColor: entity.color || '#6b7280' }} />`
      - Center column (flex-1 min-w-0):
        - `<div className="text-sm font-mono text-foreground truncate">{entity.name}</div>`
        - `<div className="text-xs font-mono text-muted-foreground truncate">{entity.leadAgent ? entity.leadAgent.name : 'No lead assigned'}</div>`
        - Last message preview: find last chatMessage where `conversation_id === entity.conversationId`, render `<div className="text-xs text-muted-foreground/60 truncate">{msg.content}</div>`
      - Right column (flex-shrink-0, flex flex-col items-end gap-1):
        - If leadAgent: `<AgentStatusBadge status={entity.leadAgent.status} variant="dot" />`
        - Timestamp of last message: `<span className="text-[10px] font-mono tabular-nums text-muted-foreground/30">{timeAgo(lastMsg.created_at)}</span>`
        - Unread dot: `<span className="w-1.5 h-1.5 rounded-full bg-primary" />` (only if unread — for now, skip unread logic as it requires tracking read state not yet implemented)
    - Selected state: `bg-surface-1 border-l-2 border-primary`
    - Hover: `hover:bg-surface-1/50`

    **teams space:**
    - Call `getTeamChatEntities(teams, departments, agentTeamAssignments, agents)`
    - Same pattern as departments but:
      - Show parent dept breadcrumb: `<span className="text-[10px] font-mono text-muted-foreground/40">{entity.departmentName}</span>`
      - Lead from team entity (resolved via agentTeamAssignments)
    - Teams without leadAgent: disabled same as departments

    **agents space:**
    - Get all agents from store
    - Split into `online` (status !== 'offline') and `offline` (status === 'offline')
    - Sort online: busy first, then idle, then error
    - Render two sections with headers: `<div className="text-[10px] font-mono uppercase tracking-wider text-muted-foreground/40 px-3 py-1">Online ({online.length})</div>` and same for Offline
    - Each agent item: name + role + status dot + preview + timestamp
    - conversationId: `agent_${agent.name}`
    - No disabled state for agents (all are clickable)

    **Container:** `<div role="listbox" id="conversation-list" aria-label="Conversations" className="flex-1 overflow-y-auto">`

    **Keyboard navigation:** onKeyDown on the listbox: ArrowUp/ArrowDown moves selection, Enter confirms. Manage activeDescendant or focus by item ref.

    **Focus management:** `useEffect` keyed on `space` — when space changes, focus the container's first item.

Implementation: `const seconds = Math.floor(Date.now()/1000) - timestamp; if (seconds < 60) return 'now'; if (seconds < 3600) return Math.floor(seconds/60) + 'm'; if (seconds < 86400) return Math.floor(seconds/3600) + 'h'; return Math.floor(seconds/86400) + 'd'`

    - If no lead agent: right side shows `<AgentStatusBadge status="offline" variant="labeled" />`

    - Find team from `teams.find(t => t.id === parsed.id)`

    **agent:**
    - Find agent from `agents.find(a => a.name === parsed.name)`
    - Left side: `[status dot via AgentStatusBadge dot variant] {agent.name}`, below: `{agent.role}` (`text-xs font-mono text-muted-foreground`)
    - Right side: `<AgentStatusBadge status={agent?.status ?? 'offline'} variant="labeled-with-queue" />`
